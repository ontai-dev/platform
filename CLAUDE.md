## Project Identity

platform is the CAPI management plane operator and the ONT-native Infrastructure
Provider for Talos. Its domain is the complete lifecycle of Talos clusters and all
tenant coordination. API group: platform.ontai.dev. CAPI infrastructure API group:
infrastructure.cluster.x-k8s.io.

**What it is:**
For target cluster lifecycle — a CAPI composition operator. It creates and owns CAPI
objects (Cluster, TalosControlPlane, MachineDeployment, SeamInfrastructureMachine) as
children of the ONT TalosCluster CR. The Seam Infrastructure Provider reconcilers within
this operator implement the CAPI InfrastructureCluster and InfrastructureMachine
contracts, delivering machineconfigs to pre-provisioned nodes on port 50000 via talos
goclient.

For management cluster bootstrap — a thin Job submitter. The direct ONT bootstrap path
via conductor Jobs remains unchanged. CAPI is not involved.

For operational Talos concerns with no CAPI equivalent — a direct Job submitter. Seven
CRDs (TalosBackup, TalosEtcdMaintenance, TalosPKIRotation, TalosRecovery,
TalosHardeningApply, TalosNodePatch, TalosCredentialRotation) submit conductor
executor Jobs directly without Kueue admission.

For tenant coordination — a record-keeping reconciler. PlatformTenant, ClusterAssignment,
QueueProfile reconcilers are thin and unchanged in behavior. Seam is fully open source with no licensing tier.

**What it is not:**
A general execution engine. No shell. No subprocess invocations. No compile-mode
logic. No Kueue admission for cluster lifecycle. Not a cluster management plane that
reimplements what CAPI already does.

The talos goclient is strictly bounded to the Seam Infrastructure Provider reconcilers
(SeamInfrastructureClusterReconciler and SeamInfrastructureMachineReconciler). No other
reconciler in platform has any talos goclient access under any circumstance.

## Project Invariants

These extend the root CLAUDE.md invariants for this component specifically.

CP-INV-001 — The talos goclient is restricted to the Seam Infrastructure Provider
reconcilers only: SeamInfrastructureClusterReconciler and SeamInfrastructureMachine
Reconciler. Every other reconciler in platform has zero talos goclient access.
This replaces the previous blanket prohibition and defines the precise boundary.

CP-INV-002 — All reconcilers outside the Seam Infrastructure Provider observe cluster
and node state exclusively through CAPI Machine status conditions, Kubernetes node
labels, and TalosCluster status. No direct Talos API queries outside the provider.

CP-INV-003 — RunnerConfig is generated by the operator using the shared runner library
for all operational Job CRDs. Never hand-coded. RunnerConfig is not generated for
CAPI-managed lifecycle operations — CAPI has no RunnerConfig concept.

CP-INV-004 — platform creates tenant namespaces. It is the sole namespace creation
authority. No other component creates seam-tenant-{cluster-name} namespaces.

CP-INV-005 — ClusterAssignment deletion never triggers cluster destruction, pack
deletion, or RBAC removal. It removes the binding record only.

CP-INV-006 — TalosClusterReset requires ontai.dev/reset-approved=true annotation before
any reconciliation proceeds. The sequence includes CAPI Cluster deletion before the
cluster-reset runner Job is submitted. The annotation gate is not bypassed by the
CAPI deletion step.

CP-INV-007 — Leader election is required. Lease name: platform-leader. Lease
namespace: seam-system. No reconciliation loop runs before leader election wins.

CP-INV-008 — TalosCluster owns all CAPI objects for target clusters via ownerReference.
No CAPI Cluster, TalosControlPlane, MachineDeployment, SeamInfrastructureCluster, or
SeamInfrastructureMachineTemplate object exists in a tenant namespace without a
TalosCluster ownerReference.

CP-INV-009 — Every TalosConfigTemplate created by this operator includes
cluster.network.cni.name: none and the Cilium-required BPF kernel parameters. No
TalosConfigTemplate is ever created without these patches. Omitting them would leave
nodes permanently NotReady.

CP-INV-010 — Kueue is not used for any operation in platform. Operational runner
Jobs submit directly without Kueue admission. Cluster lifecycle operations are
CAPI-reconciled. The Kueue ClusterQueue provisioned by guardian from QueueProfile
governs wrapper pack-deploy Jobs exclusively.

CP-INV-011 — The Seam Infrastructure Provider binary is distroless. It contains talos
goclient and kube goclient only. No shell, no system tools, no additional binaries.
This is the same constraint as conductor. INV-022 applies.

CP-INV-012 — platform is installed after guardian reaches operational state
and its own RBACProfile reaches provisioned=true. The enable phase installOrder
enforces this. No platform reconciler starts before this gate.

CP-INV-013 — The CiliumPending condition on TalosCluster is not a degraded state.
It is the expected state between CAPI cluster Running and Cilium PackInstance Ready.
No alert or degraded condition is raised during this window. MachineHealthCheck
tolerates NotReady nodes during this window via its configured tolerance period.

## Session Protocol Addition

After the standard root session protocol Steps 1–4:

Step 4a — Read platform-design.md in this repository.

Step 4b — Determine which category the target CRD belongs to before implementing
any reconciler:
- CAPI-managed lifecycle: TalosCluster (target cluster path), SeamInfrastructureCluster,
  SeamInfrastructureMachine. These reconcilers interact with CAPI objects and the ONT
  Infrastructure Provider. RunnerConfig is not generated. No runner capability is
  confirmed.
- Operational runner Job CRDs: TalosBackup, TalosEtcdMaintenance, TalosPKIRotation,
  TalosRecovery, TalosHardeningApply, TalosNodePatch, TalosCredentialRotation,
  TalosClusterReset. Verify the named capability exists in conductor-schema.md before
  implementing Job submission. If absent: raise CapabilityUnavailable, document in
  PROGRESS.md, surface to Platform Governor before proceeding.
- Tenant coordination: PlatformTenant, ClusterAssignment, QueueProfile.
  No runner capability. No CAPI objects. Pure CR management.

Step 4c — For any Seam Infrastructure Provider session: confirm talos goclient access
is bounded to SeamInfrastructureClusterReconciler and SeamInfrastructureMachineReconciler
only. Any file importing talos goclient that is not one of these two reconcilers is
an invariant violation of CP-INV-001.
